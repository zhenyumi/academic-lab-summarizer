# Academic Lab Summarizer

## Core Rule

Keep this skill thin. Do not duplicate detailed rules from `lab-site-evidence-extraction`, `lab-publication-profile`, or `lab-profile-synthesis`. Invoke those skills or read their `SKILL.md` files by path.

## Input

Start from one known lab. Required fields:

- `lab_url`
- `lab_name`
- `pi_name`
- `institution`

Optional:

- `biomedical_relevant`: Set true when PubMed should be a required Tier 1 publication source.

## Outputs

- `lab_summary_manifest.json`: Workflow manifest with status of each step and artifact paths.
- `reports/lab-summaries/<task_id>/report.html`: Canonical user-facing HTML report.
- `reports/lab-summaries/<task_id>/report.md`: Matching Markdown report.
- `reports/lab-summaries/<task_id>/report_manifest.json`: Metadata consumed by `reports/index.html`.

Worker artifacts are written under `lab_summaries/<lab_id>/` within the run directory. User-facing reports are script-generated under `reports/lab-summaries/<task_id>/`.

## Hard Constraints

- Keep this skill thin; do not duplicate worker skill rules.
- Do not modify existing crawl artifacts except through the explicit report-generation step.
- Write outputs under `lab_summaries/<lab_id>/` only.
- Each worker skill must complete and audit before the next starts.
- Final reports must be generated by scripts from stored artifacts, not hand-written in context.
- Final responses must include `reports/index.html` and the current task's `reports/lab-summaries/<task_id>/report.html`.
- All worker artifacts must be produced by running the canonical scripts, not generated directly by the LLM. The LLM may enrich script output afterwards, but the script baseline must exist first.
- After each worker skill, run the corresponding `validate_*_artifacts.py` script. If validation fails, fix artifacts before proceeding to the next step.

## Workflow

1. **Create or read `lab_summary_input.json`**: Collect lab identity fields.
2. **Run `lab-site-evidence-extraction`**: Extract structured evidence from the lab website. Read its `SKILL.md` for detailed rules. All artifacts must be produced by running the extraction script.
3. **Validate site evidence**: Run `python lab-site-evidence-extraction/scripts/validate_lab_site_artifacts.py <artifact_dir>`. Do not proceed if validation fails.
4. **Audit site evidence**: Verify `lab_site_audit.json` passes basic quality checks before proceeding.
5. **Run `lab-publication-profile`**: Search, curate, and profile publications. Preserve the required source priority policy. Read its `SKILL.md` for detailed rules. All artifacts must be produced by running the publication profile script.
6. **Validate publication profile**: Run `python lab-publication-profile/scripts/validate_publication_profile_artifacts.py <artifact_dir>`. Do not proceed if validation fails.
7. **Audit publication profile**: Verify `publication_audit.json` passes basic quality checks before proceeding.
8. **Run `lab-profile-synthesis`**: Assess lab profile and generate position signals. Read its `SKILL.md` for detailed rules. All artifacts must be produced by running `run_lab_profile_synthesis.py`. Require `position_signals.json`, `lab_summary_assessment.json`, `lab_profile.json`, `report.md`, and `lab_summary_audit.json`.
9. **Validate synthesis artifacts**: Run `python lab-profile-synthesis/scripts/validate_lab_summary_artifacts.py <artifact_dir>`. Do not proceed if validation fails.
10. **Write `lab_summary_manifest.json`**: Record the status and artifact paths of each step.
11. **Build reports**: Run the copied or shared `build_lab_summary_html.py` report builder so the task gets `report.html`, `report.md`, `report_manifest.json`, `assets/`, and `artifacts/` under `reports/lab-summaries/<task_id>/`.
12. **Refresh the report index**: Run `build_report_index.py <workspace_dir>` so `reports/index.html` links the new lab summary report.

## Manifest Structure

```json
{
  "lab_id": "example-lab-001",
  "lab_name": "Doe Neural Development Lab",
  "pi_name": "Jane Doe",
  "institution": "Example University",
  "lab_url": "https://example.edu/doe-lab",
  "steps": [
    {
      "skill": "lab-site-evidence-extraction",
      "status": "completed",
      "audit_status": "partial",
      "artifacts": {
        "lab_site_plan.json": "lab_summaries/example-lab-001/lab_site_plan.json",
        "lab_pages.jsonl": "lab_summaries/example-lab-001/lab_pages.jsonl",
        "lab_site_evidence.jsonl": "lab_summaries/example-lab-001/lab_site_evidence.jsonl",
        "lab_site_audit.json": "lab_summaries/example-lab-001/lab_site_audit.json"
      }
    },
    {
      "skill": "lab-publication-profile",
      "status": "completed",
      "audit_status": "partial",
      "artifacts": {
        "publication_search_plan.json": "lab_summaries/example-lab-001/publication_search_plan.json",
        "publication_candidates.jsonl": "lab_summaries/example-lab-001/publication_candidates.jsonl",
        "publications.curated.json": "lab_summaries/example-lab-001/publications.curated.json",
        "publication_evidence.jsonl": "lab_summaries/example-lab-001/publication_evidence.jsonl",
        "publication_audit.json": "lab_summaries/example-lab-001/publication_audit.json",
        "research_theme_profile.json": "lab_summaries/example-lab-001/research_theme_profile.json"
      }
    },
    {
      "skill": "lab-profile-synthesis",
      "status": "completed",
      "audit_status": "partial",
      "artifacts": {
        "position_signals.json": "lab_summaries/example-lab-001/position_signals.json",
        "lab_summary_assessment.json": "lab_summaries/example-lab-001/lab_summary_assessment.json",
        "lab_profile.json": "lab_summaries/example-lab-001/lab_profile.json",
        "report.md": "lab_summaries/example-lab-001/report.md",
        "lab_summary_audit.json": "lab_summaries/example-lab-001/lab_summary_audit.json"
      }
    }
  ],
  "overall_status": "completed"
}
```

All three worker steps require `audit_status` (`pass`, `partial`, `fail`, or `skipped`). A step with `audit_status` of `pass` or `partial` allows the next step to proceed. `fail` stops downstream steps. `skipped` means the step was not run due to an upstream failure.

## Stop Conditions

- If `lab_site_audit.json` reports `fail`, stop before publication search. Set publication-profile and synthesis steps to `status: "skipped"`, `audit_status: "skipped"`, `artifacts: {}`.
- If `publication_audit.json` reports `fail`, stop before synthesis. Set synthesis step to `status: "skipped"`, `audit_status: "skipped"`, `artifacts: {}`.
- If `lab_summary_audit.json` reports `fail`, set synthesis step `status: "failed"` and overall status to `failed`.
- If any required input is missing for a worker skill, stop and report the missing input.

## Validation

```bash
python academic-lab-summarizer/scripts/validate_lab_summary_manifest.py --examples
python academic-lab-summarizer/scripts/validate_lab_summary_manifest.py /path/to/manifest_dir
```

## Template-First Execution

### Copy template runner

```bash
cp academic-lab-summarizer/templates/scripts/run_academic_lab_summarizer.py <run>/tools/
```

### Run with synthetic fixtures

```bash
python academic-lab-summarizer/templates/scripts/run_academic_lab_summarizer.py \
  --input academic-lab-summarizer/examples/lab_summary_input.sample.json \
  --out /tmp/lab-summary-test \
  --site-fixtures lab-site-evidence-extraction/examples \
  --pub-fixtures lab-publication-profile/examples \
  --synth-fixtures lab-profile-synthesis/examples

python academic-lab-summarizer/scripts/validate_lab_summary_manifest.py /tmp/lab-summary-test
```

### Run with real upstream outputs

```bash
python <run>/tools/run_academic_lab_summarizer.py \
  --input <run>/lab_summaries/<lab_id>/lab_summary_input.json \
  --out <run>/lab_summaries/<lab_id> \
  --site-fixtures <run>/lab_summaries/<lab_id> \
  --pub-fixtures <run>/lab_summaries/<lab_id> \
  --synth-fixtures <run>/lab_summaries/<lab_id>

python academic-lab-summarizer/scripts/validate_lab_summary_manifest.py <run>/lab_summaries/<lab_id>
```

The template runner reads audit_status from upstream audit files, determines step statuses based on stop conditions, and writes the manifest. Adapt it for real orchestration logic.
